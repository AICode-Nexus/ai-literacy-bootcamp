# 设计文档：Claude Code Skills 实战专题教程

## 概述

为 AI 素养训练营新增一节独立专题课程，教非技术用户从零掌握 Claude Code Skills 的使用与编写。

## 定位

- **课程类型**：独立实战专题（不编号，不纳入主课程体系）
- **前置知识**：零前置，课内自包含所有必要背景
- **目标受众**：非技术用户（不会写代码，但日常使用 AI 工具）
- **终点能力**：能从零编写一个完整可用的自定义 Skill
- **预估时长**：约 2.5 小时
- **教学风格**：讲解 ≤ 40%，实操 ≥ 60%，每个 Section 有复制即用模板

## 课程结构

### Section 1：概念引入 — Skill 是什么（15 min）

**目标**：让学员理解 Skill 的本质和价值。

**核心类比**：Skill = 菜谱，Claude Code = 厨师。
- 没有菜谱，厨师也能做菜，但你得每次口头描述想吃什么
- 有了菜谱，一句"做这道菜"就够了，结果稳定、可复用

**内容要点**：
1. 什么是 Claude Code（一句话：终端里的 AI 助手，能读写文件、执行命令）
2. 什么是 Skill（一句话：预写好的指令集，让 Claude Code 按特定方式完成特定任务）
3. Skill 解决的痛点：
   - 每次都要重复输入长提示词
   - 输出质量不稳定
   - 复杂工作流记不住步骤
4. Skill vs 普通提示词的区别（对比表）

**产出**：学员能用自己的话解释"Skill 是什么"。

---

### Section 2：体验现成 Skill（20 min）

**目标**：零配置快速体验，建立直观感受。

**前置准备说明**：
- 确认已安装 Claude Code（提供安装指引链接）
- 确认终端可正常运行

**实操步骤**：
1. 打开终端，启动 Claude Code
2. 使用 `/` 查看可用 Skill 列表
3. 体验 2-3 个内置/已安装 Skill，例如：
   - `/init` — 初始化项目文档
   - `/review` — 代码审查（用一个示例仓库）
   - 一个自定义安装的 Skill（如翻译类）
4. 观察 Skill 的输入输出模式

**教学重点**：
- 调用 Skill 就像点菜：`/skill-name` + 可选参数
- 不同 Skill 有不同的触发方式和输出格式
- 让学员感受"一个斜杠命令完成原本需要 5 分钟描述的任务"

**产出**：学员成功调用至少 2 个 Skill 并看到输出。

---

### Section 3：解剖一个 Skill（25 min）

**目标**：理解 Skill 文件的结构，消除"这是代码"的恐惧。

**核心类比**：Skill 文件 = 一份结构化的工作说明书，用的是 Markdown（和写笔记一样的格式）。

**选取一个简单 Skill 作为解剖对象**（如 baoyu-translate 或自建的简单示例）。

**逐块讲解**：

```
一个 Skill 文件的组成：

┌─────────────────────────────────┐
│  文件头（元信息）                │  ← 名字、描述、触发条件
├─────────────────────────────────┤
│  指令正文                        │  ← 告诉 AI 该怎么做
├─────────────────────────────────┤
│  输出格式要求                    │  ← 结果长什么样
├─────────────────────────────────┤
│  示例（可选）                    │  ← 给 AI 看的参考
└─────────────────────────────────┘
```

**逐字段解释**：
1. 文件位置：`~/.claude/skills/` 或项目内 `.claude/skills/`
2. 文件格式：`.md` 文件（Markdown，纯文本）
3. 关键组成部分：
   - 标题和触发说明（什么时候用这个 Skill）
   - 指令正文（AI 应该做什么、怎么做）
   - 约束条件（不该做什么）
   - 输出格式（结果的结构）

**实操**：学员打开一个真实 Skill 文件，逐段标注"这是什么"。

**产出**：学员能指出 Skill 文件中每个部分的作用。

---

### Section 4：改造现成 Skill（25 min）

**目标**：通过小修改建立"我能改"的信心。

**改造任务（循序渐进）**：

**第一步：改文字**
- 打开一个现成 Skill
- 修改输出语言（如英文→中文）
- 修改输出格式（如列表→表格）
- 保存，测试效果

**第二步：改行为**
- 添加一个约束条件（如"不超过 200 字"）
- 修改工作流步骤（如增加一个"总结"步骤）
- 保存，测试效果

**第三步：改触发条件**
- 修改 Skill 的描述，让它在不同场景下被触发
- 测试新的触发方式

**教学重点**：
- 改完立刻测试，形成"改→测→看效果"的循环
- 强调：改坏了不要紧，Skill 文件随时可以恢复
- 每次只改一个地方，确认效果后再改下一个

**产出**：学员完成至少 2 处修改并验证效果。

---

### Section 5：从零创建 Skill（40 min）

**目标**：从空白文件开始，写出一个完整可用的 Skill。

**教学方法**：提供 3 个场景模板，学员选一个跟着做。

#### 可选案例 A：周报生成 Skill

**场景**：每周五把本周工作整理成结构化周报
**Skill 功能**：
- 读取指定目录下的工作记录/git log
- 按"本周完成 / 下周计划 / 需要协助"三段式输出
- 自动控制篇幅在 300 字以内

#### 可选案例 B：会议纪要整理 Skill

**场景**：把会议录音转写的文字整理成结构化纪要
**Skill 功能**：
- 输入：粘贴的会议转写文本
- 输出：参会人、议题、决议、待办事项（含负责人和截止日期）
- 自动过滤口水话和重复内容

#### 可选案例 C：内容改写 Skill

**场景**：把一篇文章改写为不同平台的风格
**Skill 功能**：
- 输入：原始文章
- 参数：目标平台（公众号 / 小红书 / LinkedIn）
- 输出：适配目标平台风格和字数的改写版本

**统一教学步骤（无论选哪个案例）**：

```
Step 1：明确需求
  → 这个 Skill 要解决什么问题？
  → 输入是什么？输出是什么？

Step 2：创建文件
  → 在正确位置创建 .md 文件
  → 写好文件头（名称、描述、触发条件）

Step 3：编写指令正文
  → 用自然语言描述 AI 应该做什么
  → 分步骤写清楚流程

Step 4：定义输出格式
  → 明确输出的结构和样式

Step 5：添加约束和边界
  → 什么不该做、什么情况下该拒绝

Step 6：测试验证
  → 调用 Skill，检查输出是否符合预期
  → 根据结果微调
```

**提供完整模板**：每个案例给出"填空式"模板，学员只需填入自己的具体内容。

**产出**：学员写出一个完整的 Skill 文件并成功调用。

---

### Section 6：测试、迭代与分享（15 min）

**目标**：掌握 Skill 的调试方法和分享方式。

**调试技巧**：
1. 常见问题排查清单：
   - Skill 没被识别？→ 检查文件位置和命名
   - 输出不符合预期？→ 检查指令是否足够具体
   - AI 忽略了某些约束？→ 用更强的措辞或加 IMPORTANT 标记
2. 迭代方法：每次只调一个变量，对比前后效果

**分享方式**：
1. 文件分享：直接把 .md 文件发给同事
2. Git 仓库：放在项目的 `.claude/skills/` 目录，团队共享
3. 社区分享：发布到 GitHub 或相关社区

**进阶方向预告**（不展开，仅告知存在）：
- Skill 可以引用其他 Skill
- Skill 可以配合 MCP Server 获得更多能力
- Skill 可以设置复杂的条件逻辑

**产出**：学员掌握调试方法，知道如何分享自己的 Skill。
